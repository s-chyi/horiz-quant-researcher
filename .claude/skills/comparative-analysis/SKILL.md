# 比较分析

## 适用场景

用户需要在多个投资标的之间做选择，需要系统性横向对比。

## 核心原则

**比较不是简单的指标罗列，而是要在同一框架下评估不同标的的「性价比」和「确定性」。比较的目的是做出取舍，不是平均分配。**

## 执行步骤

### 步骤 1: 比较框架设定

```
比较基本信息:

比较标的:
├── 标的A: [公司名] ([代码]) — 市值 [X]亿, 行业: [X]
├── 标的B: [公司名] ([代码]) — 市值 [X]亿, 行业: [X]
├── 标的C: [公司名] ([代码]) — 市值 [X]亿, 行业: [X]
└── (如有更多标的)

比较目的: [选股/行业配置/替代标的寻找/...]
比较框架: [同行业横比/跨行业比较/产业链上下游比较]

可比性评估:
├── 业务相似度: [高/中/低]
├── 市值量级: [相近/差异大]
├── 发展阶段: [相近/差异大]
└── 可比性总评: [高度可比/基本可比/可比性有限(需注意差异)]
```

### 步骤 7: ASCII 雷达图对比

```
示例:

  标的对比雷达图

         盈利能力
           10
            │
     8 ─ ─ ─┼─ ─ ─ 9
    ╱        │        ╲
  成长性 ────┼──── 估值性价比
    ╲        │        ╱
     7 ─ ─ ─┼─ ─ ─ 6
            │
        风险控制

  ── 标的A (总分: 7.8)
  ─ ─ 标的B (总分: 7.2)
  · · 标的C (总分: 6.5)

  结论: 标的A在成长性和估值上均占优，
       标的B盈利能力最强但估值偏贵，
       标的C各维度均无明显优势。
```
