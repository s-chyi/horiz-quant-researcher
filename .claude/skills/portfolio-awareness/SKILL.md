# 持仓感知分析

## 适用场景

用户提到自己的持仓、组合、仓位，并询问某个事件/趋势对其持仓的影响。
这是与 PaiPai 同题对决中**最核心的差异化能力**——PaiPai 是无状态 SaaS，永远不知道用户持什么仓。

## 核心原则

**你不是在分析"市场"，你是在分析"他的钱"。**
- 每一条建议都必须落到具体标的和具体仓位
- 不说"能源板块利好"，说"你的中海油（12%仓位）在油价100美元下预计利润增幅80%"
- 不说"建议关注风险"，说"你的隆基绿能（5%仓位）是组合中最大风险敞口，建议设-15%止损"

## 前置条件

持仓数据通过以下方式获取（优先级递减）：
1. 系统预加载的持仓 JSON（私有化部署后自动对接 OMS）
2. 用户在对话中提供的持仓信息
3. 历史对话中记录的持仓（Memory 系统）

如果没有持仓数据，主动询问：
> "我需要了解你的当前持仓才能给出针对性建议。能告诉我你持有哪些标的和大致仓位比例吗？"

## 执行步骤

### 步骤 1: 持仓解析与分类

```
持仓概览:

总规模: [X] 亿元
持仓数量: [X] 只
现金比例: [X]%

按事件敏感度分类:
├── 🟢 直接受益 ([X]% 权重): [标的列表]
├── 🟡 间接/中性影响 ([X]% 权重): [标的列表]
├── 🔴 直接受损 ([X]% 权重): [标的列表]
└── ⚪ 无关 ([X]% 权重): [标的列表]
```

### 步骤 2: 逐标的影响分析

**对每个持仓标的，必须回答：**
1. 该事件对这个标的的影响方向和幅度（定量估算）
2. 当前仓位是否合理（偏高/合理/偏低）
3. 具体操作建议（持有/加仓/减仓/清仓 + 目标仓位）
4. 关键触发条件（什么情况下改变建议）

**数据获取（每个标的）：**
- AKShare: `/api/v1/quote/{code}` — 当前价格/PE/PB/市值
- AKShare: `/api/v1/financials/{code}` — 最新财务数据
- AKShare: `/api/v1/fund-flow/{code}` — 近期资金流向
- horiz-datasource-api: `/api/v1/roadshows/search?q={公司名}` — 最新路演纪要

```
逐标的分析:

## [股票名称] ([代码]) — 仓位 [X]%

影响判定: 🟢 直接受益 / 🟡 间接影响 / 🔴 直接受损

定量估算:
├── 事件前预期: [利润/营收/估值预期]
├── 事件后预期: [利润/营收/估值预期]
├── 变化幅度: [+/-X]%
└── 对组合贡献: [+/-X] bps

仓位评估:
├── 当前仓位: [X]%
├── 建议仓位: [X]%
├── 操作: [持有/加仓至X%/减仓至X%/清仓]
└── 理由: [1-2句]

⚠️ 关注点: [如有特殊风险或注意事项]
```

### 步骤 3: 组合级别分析

```
组合整体影响:

Beta 分析:
├── 事件敏感度 Beta: [X] (组合对该事件的整体暴露度)
├── 最好情景: 组合 [+X]%
├── 基准情景: 组合 [+/-X]%
└── 最差情景: 组合 [-X]%

集中度风险:
├── 直接受益持仓占比: [X]% → [是否过度集中?]
├── 最大单一标的权重: [X]% ([标的名]) → [是否需要分散?]
└── 行业集中度: [X]% 在 [行业名] → [评估]

缺失暴露:
├── 你没有配置但应该关注的: [标的/行业] ← 理由
└── 你没有对冲但应该对冲的: [风险] ← 对冲方式
```

### 步骤 4: 综合调仓建议

```
📋 调仓建议清单 (按优先级排序):

1. [优先级最高] [操作] [标的] [从X%到Y%] ← [1句话理由]
2. [优先级次高] [操作] [标的] [从X%到Y%] ← [1句话理由]
3. ...

资金流转:
├── 减仓释放: [X] 万元 (来自: [标的列表])
├── 加仓需要: [X] 万元 (去向: [标的列表])
├── 净现金变化: [+/-X] 万元
└── 调仓后现金比例: [X]%

调仓后组合对比:
├── 事件敏感度 Beta: [调前X] → [调后Y]
├── 集中度: [改善/不变/恶化]
└── 预期 Sharpe 变化: [+/-X]
```

### 步骤 5: 投资逻辑验证（如有历史记录）

如果 Memory 系统中有用户之前的买入逻辑记录，必须做逻辑验证：

```
投资逻辑追踪:

[标的名] 买入逻辑: "[用户当时说的理由]"
├── 逻辑验证状态: ✅ 已验证 / ⚠️ 部分验证 / ❌ 已失效
├── 验证证据: [具体数据]
├── 赔率变化: 买入时 [X] → 当前 [Y]
└── 建议: [逻辑仍成立则持有, 逻辑失效则止损]
```

## 输出格式

**开头**：用 1 段话概括持仓整体受影响的方向和幅度。
**中间**：逐标的分析（按影响程度排序，最受影响的排前面）。
**结尾**：综合调仓建议清单 + 时间触发条件。

## 语言风格

参照"有态度的AI分析师"风格（见 CLAUDE.md 核心设计原则）。
- 不说"建议关注中海油"，说"你的中海油仓位 12%、在油价 100 下利润弹性约 80%，当前仓位合理，持有"
- 不说"注意风险"，说"你的隆基 5% 是组合最大风险点，跟油价无关，行业产能过剩没解决，设 -15% 止损"

## 数据源

1. AKShare API (localhost:8901) — 实时行情/财务/资金流
2. horiz-datasource-api (localhost:3001) — 路演纪要/研报
3. Web 搜索 — 最新新闻/公告
4. Memory 系统 — 用户历史持仓/买入逻辑记录
