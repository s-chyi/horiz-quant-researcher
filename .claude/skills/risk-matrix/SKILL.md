# 风险矩阵

## 适用场景

系统性识别投资标的面临的各类风险，构建概率-影响矩阵，并进行情景压力测试。

## 核心原则

**风险分析的价值不在于列出所有可能的风险（那谁都会），而在于评估概率、量化影响、设计应对。**

## 执行步骤

### 步骤 1: 风险分类扫描

```
风险全景扫描:

├── 🏢 公司特有风险 (Idiosyncratic)
│   ├── 经营风险: 产品竞争力下降、客户集中度过高、管理层变动
│   ├── 财务风险: 现金流恶化、债务到期、商誉减值、存货跌价
│   ├── 治理风险: 大股东质押、关联交易、信息披露质量
│   └── 法律风险: 诉讼、知识产权、合规处罚
│
├── 📈 行业风险 (Sector)
│   ├── 周期风险: 产能过剩、需求下滑、库存积压
│   ├── 技术替代: 新技术路线颠覆现有格局
│   ├── 竞争加剧: 新进入者、价格战、市占率流失
│   └── 政策变化: 补贴退坡、监管收紧、准入变化
│
├── 🌍 宏观/系统性风险 (Systematic)
│   ├── 经济周期: GDP 增速放缓、消费疲弱
│   ├── 货币政策: 利率上升、流动性收紧
│   ├── 地缘政治: 贸易摩擦、技术封锁、区域冲突
│   └── 汇率风险: 人民币贬值/升值
│
└── 🦢 尾部风险 (Tail Risk)
    ├── 黑天鹅: 突发事件（疫情、自然灾害、金融危机）
    ├── 监管突变: 行业整顿（如教培、互联网平台）
    └── 流动性危机: 大股东爆仓、基金踩踏
```

### 步骤 2: 概率-影响矩阵

```
概率-影响矩阵:

影响程度
    │
 极大│     [风险C]                    [风险A]
    │
  大 │              [风险D]
    │
  中 │  [风险F]                [风险B]
    │
  小 │         [风险G]    [风险E]
    │
 极小│
    └────────────────────────────────────
    极低     低      中       高     极高
                  发生概率

红色区域 (右上): 高概率+高影响 → 必须重点防范
橙色区域 (中间): 中等 → 持续监控
绿色区域 (左下): 低概率+低影响 → 了解即可
```

### 步骤 3: 关键风险深度分析

对前 3-5 个高优先级风险进行深度分析:

```
风险 #1: [风险名称]

描述: [具体描述这个风险]
概率: [X]% (依据: [...])
影响: [定量估计，如股价下跌 X%、利润减少 X 亿]

触发信号:
  ├── 早期信号: [什么迹象表明风险在升高]
  ├── 确认信号: [什么事件意味着风险已触发]
  └── 当前状态: [早期/中期/已触发/已解除]

历史类比:
  [类似风险] 在 [公司/行业] 中触发的案例:
  → 影响: [股价/业绩影响]
  → 持续时间: [多久]
  → 恢复: [是否恢复，用了多久]

应对策略:
  ├── 投资者角度: [减仓/对冲/观望]
  └── 公司角度: [公司是否有应对措施]
```

### 步骤 4: 情景压力测试

```
压力测试:

场景 1: 基准情景 (Base Case) — P=[X]%
  假设: [维持现有趋势]
  业绩影响: 净利润 [X] 亿
  估值影响: PE [X]x, 股价 [X] 元

场景 2: 乐观情景 (Upside) — P=[X]%
  假设: [最有利条件]
  业绩影响: 净利润 [X] 亿 (+[X]%)
  估值影响: PE [X]x, 股价 [X] 元 (+[X]%)

场景 3: 悲观情景 (Downside) — P=[X]%
  假设: [主要风险部分兑现]
  业绩影响: 净利润 [X] 亿 (-[X]%)
  估值影响: PE [X]x, 股价 [X] 元 (-[X]%)

场景 4: 极端情景 (Stress) — P=[X]%
  假设: [多个风险同时兑现]
  业绩影响: 净利润 [X] 亿 (-[X]%)
  估值影响: PE [X]x, 股价 [X] 元 (-[X]%)

最大回撤估计: -[X]% (极端情景)
期望收益/最大损失 比: [X]:1
```
