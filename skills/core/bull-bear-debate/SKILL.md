# 多空辩证

## 适用场景

用户想了解某个投资标的/主题的正反观点，或市场上存在明显分歧时使用。这是超越 PaiPai 最直接的技能 — PaiPai 倾向给"标准答案"，我们显式展示思辨过程。

## 核心原则

**永远不给"唯一正确答案"。投资的本质是在不确定性中做概率判断。**

## 执行步骤

### 步骤 1: 辩题定义

```
辩题: [清晰表述争议焦点]
例: "宁德时代在锂电产能过剩周期中能否维持超额收益？"

当前市场共识: [描述主流观点]
分歧点: [市场在哪些问题上意见不一]
```

  ...

### 步骤 4: 交叉诘问

**这是本技能最关键的环节 — 让多空双方互相质疑**

```
🔄 交叉诘问

多头对空头论点 1 的回应:
  "你说[空头论点]，但 [反驳/补充]，因为 [证据]"

空头对多头论点 1 的回应:
  "你说[多头论点]，但 [反驳/补充]，因为 [证据]"

[继续 2-3 轮交锋...]

交锋后的深层分歧:
  → 根本分歧在于: [本质上双方对什么的判断不同]
  → 这个分歧将由 [什么事件/数据] 来验证
  → 验证时间窗口: [大约什么时候能见分晓]
```

## 反面论证强制规则

1. **不允许一边倒**: 如果初始分析倾向一方，必须花同等篇幅论证另一方
2. **不允许稻草人**: 反方论点必须是真实的、有力的，不是凑数的弱论点
3. **必须找到深层分歧**: 不只是列论点，而是找到双方真正的认知差异
4. **必须给出验证方法**: 告诉用户如何判断哪一方更可能正确
