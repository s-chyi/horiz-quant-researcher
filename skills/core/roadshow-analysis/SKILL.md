# 路演纪要分析

## 适用场景

用户提供路演纪要/调研纪要/电话会纪要，或要求检索特定公司的路演记录，需要深度分析管理层传递的信号。

## 核心原则

**路演纪要的核心价值不是管理层说了什么（那可能是PR话术），而是：(1) 措辞变化暗示了什么，(2) 没说什么（回避了什么），(3) 能否交叉验证。**

## 执行步骤

### 步骤 1: 纪要结构化提取

首先从 horiz-datasource-api 检索相关路演纪要:

```
路演纪要基本信息:

公司: [公司名称] ([股票代码])
路演类型: [业绩说明会/投资者调研/券商策略会/行业论坛/单独路演]
日期: [路演日期]
参与方: [管理层出席人员]
组织方: [券商/机构]
数据来源: horiz-datasource-api (纪要ID: [X])
数据可靠性: 🟢/🟡 [一手纪要/二手转述]
```

### 步骤 2: 核心信息提取

```
管理层核心表态:

├── 📈 业绩相关
│   ├── 营收指引: [管理层表态] ← 原文: "[引用原文]"
│   ├── 利润指引: [管理层表态] ← 原文: "[引用原文]"
│   ├── 毛利率趋势: [管理层表态]
│   └── 费用控制: [管理层表态]
│
├── 🏭 经营动态
│   ├── 订单情况: [管理层表态]
│   ├── 产能利用率: [管理层表态]
│   ├── 客户拓展: [管理层表态]
│   ├── 新产品/新业务: [管理层表态]
│   └── 竞争格局判断: [管理层表态]
│
├── 💰 资本运作
│   ├── 分红计划: [管理层表态]
│   ├── 回购计划: [管理层表态]
│   ├── 融资需求: [管理层表态]
│   └── 并购意向: [管理层表态]
│
└── 🌍 行业/宏观判断
    ├── 行业景气度: [管理层表态]
    ├── 政策影响: [管理层表态]
    └── 出海/国际化: [管理层表态]
```

### 步骤 3: 措辞分析与信号解读

```
措辞信号分析:

├── 🟢 积极信号 (管理层释放正面预期):
│   ├── 信号1: "[原文措辞]"
│   │   └── 解读: [为什么这是积极信号]
│   ├── 信号2: "[原文措辞]"
│   │   └── 解读: [...]
│   └── 信号强度: [强/中/弱]
│
├── 🔴 消极信号 (管理层在管理预期/暗示困难):
│   ├── 信号1: "[原文措辞]"
│   │   └── 解读: [为什么这是消极信号]
│   ├── 信号2: "[原文措辞]"
│   │   └── 解读: [...]
│   └── 信号强度: [强/中/弱]
│
├── 🟡 措辞变化 (与前次路演对比):
│   ├── 变化1:
│   │   ├── 前次: "[上次路演措辞]"
│   │   ├── 本次: "[本次路演措辞]"
│   │   └── 解读: [措辞从X变为Y，暗示...]
│   ├── 变化2: ...
│   └── 措辞变化方向: [整体偏积极/偏消极/中性]
│
├── ⚫ 回避信号 (被问到但没有正面回应):
│   ├── 回避话题1: [问题] → 管理层回应: "[模糊回应]"
│   │   └── 解读: [回避通常意味着...]
│   ├── 回避话题2: ...
│   └── 回避倾向: [对XX话题明显回避]
│
└── 🔵 新增信息 (以前未提及的新表态):
    ├── 新信息1: "[内容]" → 重要性: [高/中/低]
    └── 新信息2: "[内容]" → 重要性: [高/中/低]
```

### 步骤 4: 交叉验证

```
交叉验证:

├── vs 财报数据:
│   ├── 管理层说法: [X]
│   ├── 财报实际: [Y]
│   └── 一致性: [一致/矛盾/无法验证]
│
├── vs 行业数据:
│   ├── 管理层对行业的判断: [X]
│   ├── 第三方行业数据: [Y]
│   └── 一致性: [一致/矛盾/无法验证]
│
├── vs 竞争对手路演:
│   ├── 竞争对手的说法: [X]
│   ├── 与该公司的矛盾点: [如有]
│   └── 谁更可信: [分析]
│
├── vs 该公司历史路演:
│   ├── 管理层过往指引完成率: [X]% (如可统计)
│   ├── 过往乐观/保守倾向: [偏乐观/偏保守/较准确]
│   └── 本次可信度调整: [基于历史表现调整]
│
└── 综合可信度: [高/中/低]
    └── 理由: [为什么给出这个评级]
```

### 步骤 5: 投资启示

```
投资启示:

对当前投资论点的影响:
├── 增强论点: [路演中哪些信息支持看多/看空]
├── 削弱论点: [路演中哪些信息与预期不符]
└── 新增考量: [路演揭示的之前未考虑的因素]

关键变量更新:
├── 变量1: [之前预期] → [路演后调整为] (调整原因: [...])
├── 变量2: [之前预期] → [路演后调整为]
└── 盈利预测影响: [需要上调/下调/维持] [X]%

后续跟踪:
├── 验证点1: [什么时候/什么数据可以验证管理层说法]
├── 验证点2: [...]
└── 下次路演/财报时间: [日期]
```

### 步骤 6: 多纪要纵向对比 (当检索到多篇纪要时)

```
纵向对比 (近N次路演):

                  [日期1]    [日期2]    [日期3]    趋势
营收指引          [X]        [X]        [X]        ↑/→/↓
利润指引          [X]        [X]        [X]        ↑/→/↓
行业景气判断      [X]        [X]        [X]        ↑/→/↓
管理层信心度      [X]        [X]        [X]        ↑/→/↓
关键措辞变化      [简述]     [简述]     [简述]     [总结]

纵向趋势解读:
  管理层在过去[N]次路演中，对[XX]的态度从[X]转变为[Y]，
  暗示[解读]。需要重点关注[关键变量]。
```

## 数据源

1. **horiz-datasource-api** → 核心数据源，路演纪要全文检索
   - `/api/roadshows` — 路演纪要列表和详情
   - `/api/roadshows/search` — 按公司/关键词搜索
2. Web 搜索 → 补充最新路演信息、行业数据交叉验证
3. 用户提供的纪要内容
