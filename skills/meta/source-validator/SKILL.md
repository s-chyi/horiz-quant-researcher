# 数据溯源验证器

## 功能

自动验证分析中引用的每条关键数据，标注来源和可靠性等级，防止"幻觉数据"污染分析结论。

## 数据质量等级

### 🟢 一手数据 (Primary Source) — 可直接引用

| 来源类型 | 示例 | 验证方式 |
|----------|------|---------|
| 交易所公告 | 上交所/深交所公告 | 巨潮资讯网原文 |
| 公司财报 | 年报/季报/半年报 | 交易所官网 PDF |
| 央行/统计局数据 | GDP/CPI/M2/LPR | 官网数据库 |
| 监管文件 | 证监会/银保监政策 | 官方网站 |
| 交易数据 | 行情/成交量/资金流向 | 交易所实时数据 |

### 🟡 二手数据 (Secondary Source) — 引用时标注来源

| 来源类型 | 示例 | 注意事项 |
|----------|------|---------|
| 券商研报 | 中金/中信/国盛研报 | 标注分析师和日期 |
| 行业协会数据 | 中汽协/光伏协会 | 可能有滞后 |
| 新闻媒体 | 财联社/证券时报 | 需交叉验证 |
| 第三方数据平台 | Wind/Choice/Tushare | 标注数据截止日 |

### 🔴 三手数据 (Tertiary Source) — 必须交叉验证后才可引用

| 来源类型 | 示例 | 风险 |
|----------|------|------|
| 社交媒体 | 雪球/微博讨论 | 可能有虚假信息 |
| 匿名来源 | "据知情人士透露" | 无法验证 |
| 模型估算 | AI 生成的预测数字 | 必须标注为估算 |
| 非官方统计 | 草根调研/渠道数据 | 样本偏差 |

## 验证流程

### 步骤 1: 数据标记

扫描分析文本，标记所有数字/事实声明:
```
类型 A: 精确数字 (营收 421 亿、增速 28.3%) → 必须有来源
类型 B: 定性判断 (行业景气度回升) → 需要数据支撑
类型 C: 逻辑推论 (因此估值偏低) → 需要推理过程
```

### 步骤 2: 来源匹配

为每条数据匹配来源:
```
[数据] → [来源] → [质量等级] → [时效性]
```

### 步骤 3: 缺失补全

对于无来源的数据:
- 尝试从已接入数据源查询验证
- 无法验证的数据标注为 ⚠️ 待验证
- 明确为 AI 估算的标注 🤖 模型估算

### 步骤 4: 冲突检测

当不同来源的数据冲突时:
- 优先采信🟢一手数据
- 标注冲突并展示各来源数据
- 给出采信建议和理由

## 标注格式

```markdown
公司 2025Q3 营收 421 亿元，YoY +28.3%
—— 🟢 来源: 公司季报 (2025-10-28 披露)

行业分析师平均预期 2026 年营收增速 15-20%
—— 🟡 来源: Wind 一致预期 (截至 2025-11-15, 覆盖 28 家券商)

市场传闻公司将获得某大客户订单
—— 🔴 来源: 雪球讨论 (未经证实, 2025-11-20)
```

## 反幻觉机制

以下情况触发强制检查:
1. 引用了过于精确的未来预测数字
2. 声称了无公开来源的"内部信息"
3. 使用了看似合理但实为编造的历史数据
4. 引用了已过期的数据作为当前事实

处理方式:
- 标注 ⚠️ 并说明不确定性
- 给出可验证的替代信息
- 绝不让未验证数据影响核心结论
